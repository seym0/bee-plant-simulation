# Модуль визуализации экологического симулятора

## Обзор

Модуль `visualize_dynamics.py` предоставляет инструменты для сбора данных симуляции и создания профессиональных графиков, которые демонстрируют многолетнюю динамику популяций пчел и растений, а также фенологический рассинхрон.

## Компоненты

### 1. Класс `SimulationLogger`

Логирует ключевые метрики каждый год симуляции.

**Атрибуты:**
- `years`: список номеров лет
- `bee_population`: популяция пчел по годам
- `temperature_mean`: среднегодовая температура
- `bee_activation_day`: день просыпания пчел из спячки
- `plant_data`: словарь `{plant_name: [seed_bank_values]}`

**Методы:**
- `log_year(year, bees, flora, nature)` - логирует метрики за один год
- `_calculate_activation_day(activation_temp, nature)` - вычисляет день активации пчел

### 2. Функция `plot_simulation_results(history_data, output_file)`

Создает двухпанельный дашборд:

**Верхняя панель (Популяции):**
- График популяции пчел (толстая оранжевая линия)
- Графики seed_bank для каждого вида растений (пунктирные линии разных цветов)
- Легенда, сетка, подписи осей

**Нижняя панель (Фенологический сдвиг):**
- День просыпания пчел (левая ось Y, синий график)
- Среднегодовая температура (правая ось Y, красный график)
- Демонстрирует нарастающий фенологический рассинхрон

**Параметры:**
- `history_data`: объект SimulationLogger
- `output_file`: путь для сохранения (по умолчанию `simulation_results.png`)

### 3. Функция `print_summary_statistics(history_data)`

Выводит текстовую статистику симуляции в консоль:
- Общее количество лет
- Максимальная популяция пчел и год её достижения
- Год коллапса
- Динамика каждого вида растений
- Фенологический сдвиг (в днях)
- Температурные изменения (в °C)

## Интеграция в main.py

### Шаг 1: Импорт модуля

```python
from visualize_dynamics import SimulationLogger, plot_simulation_results, print_summary_statistics
```

### Шаг 2: Инициализация логгера

В главном блоке `if __name__ == "__main__":` добавьте:

```python
# Инициализация логгера для сбора данных
logger = SimulationLogger()
```

### Шаг 3: Логирование в главном цикле

В конце каждого года (после репродукции растений) добавьте:

```python
# --- ПОДВЕДЕНИЕ ИТОГОВ ГОДА ---
bees.complete_year(sync_days)

# Репродукция растений на основе визитов пчел
for plant in flora:
    plant.reproduce()

# ✓ ЛОГИРОВАНИЕ МЕТРИК ЗА ГОД
logger.log_year(year, bees, flora, nature)

# Остальной код вывода...
```

### Шаг 4: Визуализация результатов

После завершения цикла симуляции добавьте:

```python
# ========== ВИЗУАЛИЗАЦИЯ И АНАЛИЗ РЕЗУЛЬТАТОВ ==========
print_summary_statistics(logger)
plot_simulation_results(logger, output_file='simulation_results.png')
```

## Интерпретация результатов

### Верхний график (Популяции)

- **Пик популяции пчел:** обычно наблюдается на 10-20-м году, когда условия еще благоприятны
- **Последующий спад:** отражает снижение доступности нектара из-за рассинхрона
- **Вид растений "Очень ранний" вымирает рано:** вероятно, первым страдает от фенологического рассинхрона

### Нижний график (Фенологический сдвиг)

- **Движение линии "день просыпания пчел" влево:** пчелы приспосабливаются и просыпаются раньше
- **Температурная линия растет:** показывает глобальное потепление
- **Расхождение линий:** демонстрирует растущий рассинхрон между пробуждением пчел и началом цветения растений

## Расширение модуля

Для добавления новых метрик в `SimulationLogger`:

1. Добавьте новый список в `__init__`:
   ```python
   self.new_metric = []
   ```

2. Обновите метод `log_year()`:
   ```python
   self.new_metric.append(value)
   ```

3. Используйте в функции `plot_simulation_results()` для новых графиков.
